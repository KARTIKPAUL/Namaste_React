2. Difference between Library and Framework ?
Ans : The main difference between Library and Framework is Inversion of control.When using a library the controls remains with the developer who tells the application when to call library functions. When using a framework , the control is reversed , which means that the framework tells the developer where code needs to be provided and calls it as it requires.

3.What us CDN ? why do we use it ?
Ans : CDN stands for Content Delivery Network(CDN).A CDN is a network of servers spread across the different locations around the world that deliver files to the user.|
Instead of loading files one single server far away, browser downloads them from the nearest server.

4.Why React is known is React ?
Ans : React is a Javascript Library. The name 'React' was chosen because the library was designed to allow developers to react to changes in state and data within the application, and to update the user interface in a declarative and origin than the hostig efficient manner.

5.What is cross origin in script tag ?
Ans : The cross origin attribute in the script tag enables Cross-Origin Resource Sharing(CORS) for loading external Javascript files from different origin than the hosting web page.This allows us the script to access resources from the server hosting tha script, such as making HTTP requests or accessing data.

6.What is the difference between React and ReactDOM ?
Ans : React is the core Javascript library for building user interface amd managaing component logic and state,while react-dom is a supplementary library that specifically handles the rendering and updating of those React components in a web browser's Document Object Model(DOM)

7.What is difference between react.development.js and react.production.js files via CDN?
Ans : The core difference between React development and production files available via a Content Delivery Network(CDN) is that the development file (react.development.js) is optimized for debugging and developer experience, while the productions file(react.production.min.js) optimized for performance and minimal file size for the end user.
